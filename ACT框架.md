# ACT 框架设计

## 🛠️ 技术层 ACT：基础设施

### A - AI能力（四类能力）
- **决策**：病情互动讨论/建议/决策支持 （ai-mdt）
- **生成**：知识库问答（小x宝助手）/ 个体化宣教（科普视频/文章/博客）
- **检索**：知识专题研究 / 规范指南定位 / FAQ 召回 （knows/get笔记）
- **匹配**：医患对接 / 同伴推荐 / 资源链接（待完善）

> 以上能力以下列开源形态统一交付：
> - API-hub：开放服务 API
> - 智能体套件：Prompt模板集/RAG 工作流/智能体脚本
> - 定制化模型：med-gamma3，小馨宝llm等微调模型权重
> - 知识星云：多种开放知识库连接网络

### T - Toolkits（场景工具箱）
- 🧰 肿瘤管理决策支持工具包（AI助手+新手宝典）
- 🧘 心理支持工具包（情绪识别+CBT 话术+冥想音频）
- 📋 症状管理工具包（监测+预警+复诊规则）
- 🤝 照护者工具包（护理知识+互助脚本+资源链接）
- 形态：100+轻应用+知识/操作经验模版

### C - Community（社区平台）
- 🌐 线上社区空间
- 📞 线下支持小组
- 🏢 医疗机构联盟
- 👨‍👩‍👧‍👦 家属互助网络
- 形态：蓝马甲志愿者/覆盖全周期需要的病友群和指引路径/社区直播/心理支援小组/工作坊

## ❤️ 情感层 ACT：心理基石

### A - Acceptance（接受）
- 🫂 接受疾病现实
- 💭 接受情绪波动
- 🤗 接受社区支持
- 🎭 接受脆弱一面

### C - Commitment（承诺）
- 💪 承诺相互陪伴
- 🎯 承诺价值导向行动
- 🔄 承诺持续支持
- 🌟 承诺寻找意义

### T - Trust（信任）
- 🤝 信任社区的真诚
- 🩺 信任医疗的专业
- 💝 信任同伴的善意
- 🛡️ 信任信息的可靠性

## 🔗 双重 ACT 的协同效应

> 技术层依赖关系：**AI 能力 → 工具箱 → 入口**，确保能力复用与场景解耦。

### 技术层支撑情感层
- **AI 能力 + Acceptance**：智能算法帮助患者接受现实
- **Toolkits + Commitment**：场景资源支持承诺的行动
- **Community + Trust**：平台建立信任的连接

### 情感层指导技术层
- **Acceptance 指导 AI 能力设计**：不逼迫正面思考的算法伦理
- **Commitment 驱动 Toolkits**：基于价值的功能组合
- **Trust 规范 Community**：安全、包容、支持性的社区文化

## 💎 “轻连接、重支持”在双重 ACT 中的体现

### 轻连接 = 技术层的高效工具
- AI 驱动的轻应用降低参与门槛
- 精准匹配的工具包提高效率
- 开放包容的 Community 扩大连接范围

### 重支持 = 情感层的深度连接
- Acceptance 提供情感安全空间
- Commitment 创造持续支持关系
- Trust 建立可靠的支持基础
