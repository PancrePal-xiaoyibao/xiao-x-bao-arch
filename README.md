# 小X宝架构仓库

> 让AI消除肿瘤患者信息差,通过开源社区的力量惠及更多患者

本仓库由小x宝架构志愿者团队创建、维护和开放。欢迎协助社区推动架构进化,让更多肿瘤、罕见病、慢性病患者/家属,可以得到科技普惠,能够在社区协助下,成为病情管理的主人,并积极获益。

我们希望,志愿者的努力,能够通过技术实现,不断延伸。

## 📌 项目定位

小X宝是一个**肿瘤患者AI智能支持开源社区项目**,聚焦于)2025-2026年技术架构规划,涵盖业务、应用与技术三层架构设计。

### 核心研究方向
- 🤖 **多智能体协作**:研究和实践多智能体系统在医疗场景的应用
- 📊 **LLM性能评估**:建立统一的LLM/RAG测评框架和指标体系
- 🏥 **全生命周期AI应用**:患者护理支持、病情管理、心理支持等
- 🌐 **开源社区框架建设**:采用ACT框架指导系统设计和社区运营

## 🏗️ 架构体系

### 三层架构设计

```
业务架构层 → 应用架构层 → 技术架构层
    ↓           ↓           ↓
患者需求    AI智能体     RAG/LLM引擎
社区运营    轻应用       知识库星云
资源整合    小程序       多模型支持
```

### ACT框架

项目采用双重ACT框架设计:

**技术层ACT**:
- **A - AI能力**:决策、生成、检索、匹配四类核心能力
- **C - Community**:线上社区空间、线下支持小组、医疗机构联盟
- **T - Toolkits**:肿瘤管理工具包、心理支持工具包、症状管理工具包

**情感层ACT**:
- **A - Acceptance**:接受疾病现实、情绪波动、社区支持
- **C - Commitment**:承诺相互陪伴、价值导向行动、持续支持
- **T - Trust**:信任社区真诚、医疗专业、同伴善意
